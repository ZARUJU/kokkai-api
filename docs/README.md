# docs

配布データの仕様と、そのデータがどこからどのように作られるかを、データ種別ごとに整理した文書を置く。

## 一覧

- `kaigiroku_data.md`
  会議録データ。国会会議録検索システム API から `data/kaigiroku/` をどう生成するかをまとめる
- `gian_data.md`
  議案データ。衆議院サイトの議案一覧・審議経過・本文情報から、`data/gian/` をどう生成するかをまとめる
- `seigan_data.md`
  請願データ。衆参両院サイトの請願一覧・個票から、`data/seigan/` をどう生成するかをまとめる
- `shitsumon_data.md`
  質問主意書データ。衆参両院サイトの一覧・個票から、`data/shitsumon/` をどう生成するかをまとめる
- `people_data.md`
  人物インデックス。議案・請願・質問主意書・会議録の配布データから、`data/people/index.json` をどう生成するかをまとめる

## 方針

- 取得元 URL または入力ファイル
- 中間生成物
- 最終的な配布 JSON の配置と項目

の 3 点を、各データ種別ごとに同じ文脈で追える形にする。
